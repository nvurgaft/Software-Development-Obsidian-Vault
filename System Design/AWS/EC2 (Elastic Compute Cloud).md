Virtual machines on the cloud. You create an instance with a desired OS and use SSH to connect. 
Using the [[AMI (Amazon Machine Image)]], the user chooses an OS for the virtual machine, spins it up and has full control over it.
Not a container, or a part of an Elastic cluster, but can be configured as one.
Used for simple or legacy apps, testing, demos, simple workstations, storage, highly customizable.

Payment is by use time of active users, hence the "Elastic". 

* Can easily spin up and terminate instances
* Simple 
* Pay as your go
* Accessed using a terminal using ssh
* If the address is published, it can be accessed via the Internet
