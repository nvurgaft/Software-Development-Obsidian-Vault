Black Box Testing is a software testing technique where the tester evaluates the functionality of an application without knowing its internal code structure. It focuses on inputs and expected outputs based on requirements.

Testing API's for various values and results, testing validation of inputs, testing expected behavior are all types of black box testing.

### Advantages

1. Black box testing can be automated or manual.
2. Black box testing can be performed on a compiled or deployed instance of your code.
3. Black box testing allows testing by people who are not familiar with the codebase and forgo any need to interact with it.

### Disadvantages

1. Requires more setting up.
2. In case of errors it is harder to pin point the point of failure. So additional tools such as logs and monitoring systems are used.
